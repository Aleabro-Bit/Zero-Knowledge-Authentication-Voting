# Zero-Knowledge-Authentication-Voting

This repository contains the source code developed for my Bachelor's degree thesis. 

The purpose of this project is to demonstrate the applicability of Zero-Knowledge proofs (zkSNARKs) on the Ethereum blockchain to create a decentralized, secure, and privacy-preserving e-voting system. Through this system, users can mathematically prove they are of legal age and eligible to vote without ever revealing their actual personal data in plaintext.

---

## Repository Structure

The project consists of cryptographic circuits, smart contracts, and utility scripts:

* **`VerifiedVoting.sol`**: The main smart contract that manages the decentralized voting logic. It allows only users who have successfully passed the ZK verification to cast a vote. It automatically handles transitions between four electoral states (Open, Runoff, Concluded, Cancelled) based on time deadlines provided during deployment.
* **`age_check.zok`**: The logic circuit written in ZoKrates. It receives the user's age, an identity credential ID, and a user secret (as cryptographically private inputs), along with current/expiration timestamps, an expected attestation hash, and the wallet address (as public inputs). It mathematically guarantees that the user is 18+, the credential is valid, the cryptographic signature matches, and binds the proof to the caller's wallet.
* **`verifier.sol`**: The verification smart contract automatically generated by ZoKrates. It contains the mathematical logic and the hardcoded Verification Key necessary to validate the zkSNARK proofs on-chain.
* **`input_builder.js`**: A utility script that handles the off-chain cryptographic logic. It generates Unix timestamps, computes the expected attestation hash, formats the Ethereum address into decimal form, and builds the exact CLI command required to compute the ZoKrates witness.
* **`extract.hs`**: A formatting script that reads the `proof.json` file generated by ZoKrates and extracts the data into the correct format required by the Solidity verification contract.

## Usage and Setup

To properly run the authentication and voting system, you must compile the circuit and generate the cryptographic keys.

> **WARNING: Key Regeneration (Trusted Setup)**
> To successfully validate proofs, the ZoKrates `setup` phase **must** be executed locally. The setup command generates a Proving Key (to create proofs) and a Verification Key (to validate them). 
> Every time the setup is performed, it is imperative to export the new `verifier.sol` contract and deploy it (or link it) so that `VerifiedVoting.sol` has the updated cryptographic keys. Proofs generated with outdated keys or keys that are not synchronized with the smart contract will systematically fail.

### Step-by-Step Workflow:

**Circuit Compilation and Setup**
```
# Compile the circuit
zokrates compile -i age_check.zok

# Run the trusted setup to generate the keys (verifying.key and proving.key)
zokrates setup

# Export the verification smart contract (generates verifier.sol)
zokrates export-verifier
```
**Witness and Proof Generation**
Run the `input_builder.js` script to compute the necessary hashes and timestamps. The script will output the exact command you need to run. Execute the generated command, which follows this structure:
```
zokrates compute-witness -a <AGE_USER> <CREDENTIAL_ID> <USER_SECRET> <CURRENT_TIMESTAMP> <EXPIRATION_TIMESTAMP> <EXPECTED_ATTESTATION_HASH> <DECIMAL_ADDRESS>

# Generate the proof.json file
zokrates generate-proof
```
**Smart Contract Verification**
Run the `extract.js` script on the generated proof.json file to format the A, B, C parameters and the public input array.

Submit a transaction to the verification and registration function of the `VerifiedVoting.sol` contract, including the extracted parameters.

If the authentication is successful, your address will be permanently authorized to vote as long as the election remains in the Open or Runoff state.
